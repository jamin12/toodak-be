### Member 도메인
- `id: MemberId` — `@JvmInline value class MemberId(val value: UUID)`, UUID v7
- `email: Email` — `@JvmInline value class Email(val value: String)`, **unique 제약 X**
- `status: MemberStatus` — `ACTIVE | WITHDRAWN`
- `createdAt: Instant`
- `withdrawnAt: Instant?`
- **private constructor** + companion object:
  - `register(email: Email): Member` — 신규 가입 (id = UUID v7 생성, status = ACTIVE)
  - `restore(id, email, status, createdAt, withdrawnAt?): Member` — JPA에서 복원
- 메서드: `changeEmail(newEmail)`, `withdraw()` (시그니처만 — 1차 미구현)

### SocialIdentity (Member 1:N)
- `provider: Provider` — enum (GOOGLE, KAKAO, APPLE …), `trustsEmailVerification: Boolean` 플래그 보유
- `providerUserId: ProviderUserId` — `@JvmInline value class`
- `emailVerifiedAt: Instant?` — 자동 연결 정책 판단용
- `linkedAt: Instant`
- unique: `(provider, providerUserId)` (DB unique 제약)
- **private constructor** + companion object:
  - `link(memberId, provider, providerUserId, emailVerifiedAt?): SocialIdentity` — 신규 연결
  - `restore(...)`: 복원

### RefreshToken (auth 컨텍스트)
- `id: RefreshTokenId`, `memberId: MemberId`
- `tokenHash: String` — DB엔 해시만 저장 (평문은 응답에만)
- `deviceId: DeviceId`, `deviceLabel: String?`
- `issuedAt, expiresAt: Instant`
- `revokedAt: Instant?`, `rotatedFromId: RefreshTokenId?`
- **private constructor** + companion object:
  - `issue(memberId, deviceId, deviceLabel, ttl): IssuedRefreshToken` — 발급 (plain + entity)
  - `restore(...)`: 복원
- 메서드: `revoke(at)`, `rotate(at, ttl) → IssuedRefreshToken`

### GoogleUser (auth 컨텍스트의 도메인 모델)
- `sub: String`, `email: String`, `emailVerified: Boolean`, `name: String?`, `picture: String?`
- GoogleOAuthOutPort가 검증된 결과를 이 타입으로 반환

## OutPort 목록 (5개) — "한 외부 의존 = 한 OutPort" 규칙
| OutPort | 외부 의존 | 메서드 |
|---------|-----------|--------|
| `MemberOutPort` | DB · member 테이블 | `save`, `findById`, `findByEmail` |
| `SocialIdentityOutPort` | DB · social_identity 테이블 | `save`, `findBy(provider, providerUserId)` |
| `RefreshTokenOutPort` | DB · refresh_token 테이블 | `save`, `findByHash`, `findActiveBy(memberId, deviceId)`, `revokeAllByMember` |
| `GoogleOAuthOutPort` | Google JWKS (HTTP) | `verifyIdToken(idToken): GoogleUser` |
| `JwtOutPort` | JJWT 라이브러리 | `issueAccessToken(memberId, ttl)`, `parseAccessToken(token): AccessTokenClaims` |

## 도메인 불변식 (코드/리뷰 기준)
- Member는 반드시 1개 이상의 SocialIdentity로 가입한다 (Member.register만 노출)
- `Member.id`는 한 번 부여되면 영원히 불변
- `(provider, providerUserId)`는 시스템 전체에서 unique
- `Member.email`은 **변경 가능**, unique 제약 없음
- `MemberStatus == WITHDRAWN`이면 로그인 불가 (UseCase에서 거부)
- 자동 계정 연결은 `provider.trustsEmailVerification == true` 이면서 `google.emailVerified == true` 일 때만 허용
- RefreshToken은 항상 특정 Member에 귀속
- RefreshToken 평문은 DB에 저장 X — `tokenHash`만 저장

## 제약 조건
- 모바일 클라이언트 우선 → ID Token Flow만 지원 (Authorization Code Flow 미사용)
- 외부 출시 계획 있음 — 약관/PII 처리는 별도 컨텍스트로 추후 추가
- DB 종류 미정 — 운영 PostgreSQL 가정, 로컬은 H2 권장
- 재가입 정책 보류 — 1차에선 탈퇴 기능 자체 미구현
- Spring Security 사용 — JWT 필터 체인 직접 구성

## 주의사항 (실수하기 쉬운 부분)
- ⚠️ **이메일 unique 제거** — DB DDL에 unique 인덱스 걸지 말 것. 동시성 race는 `(provider, providerUserId)`로 막힘.
- ⚠️ **자동 연결 emailVerified 체크 잊지 말 것** — Google은 항상 verified지만, Kakao 추가 시 문제 발생. `Provider.trustsEmailVerification` 분기 필수.
- ⚠️ **RefreshToken hash 저장** — 평문 저장 금지. 클라에 평문 반환은 발급 시점에만.
- ⚠️ **탈취 감지 로직** — 이미 revoked된 토큰으로 refresh 시도가 들어오면 **해당 Member의 모든 RefreshToken을 revoke**. 단순히 "Invalid" 거부만 하면 안 됨.
- ⚠️ **Access Token은 즉시 무효화 불가** — JWT stateless 특성. 짧은 TTL(15~30분)로 가야 함.
- ⚠️ **Member.register 외에 Member 생성 경로 차단** — `private constructor` 또는 패키지 가시성 활용.
- ⚠️ **비즈니스 로직은 UseCase에서** — Controller는 얇게, Adapter는 변환만.
- ⚠️ **Boolean 접두사 금지** — code-convention 스킬 참조 (`isXxx`, `hasXxx` 같은 접두사 X).

## 스킬 규칙 요약 (코드 작성 시 항상 준수)

### 헥사고날 아키텍처 (`architecture`)
- 의존성 방향: `adapter → application → domain` 단방향
- domain에 Spring/JPA 어노테이션 금지
- application은 adapter import 금지 (OutPort 인터페이스로만 통신)
- 새 기능 생성 순서: Domain Model → InPort → OutPort → UseCase → JPA Entity/Adapter → REST Controller

### 도메인 모델 (`domain-model`)
- **private constructor + companion object** 필수
- 팩토리 메서드: `create()`/`register()`/`issue()` (신규) + `restore()` (복원) **둘 다 제공**
- `val` 우선, 변경 필요한 필드는 `private set`
- 비즈니스 로직(검증/계산/상태 전이)은 도메인 모델 안에 위치
- VO: `@JvmInline value class`, `domain/{도메인}/vo/` 위치, `init`에서 형식 검증

### UseCase (`usecase`)
- UseCase는 OutPort 인터페이스만 의존 (Repository/Adapter 직접 의존 금지)
- InPort 반환 타입: **Result DTO** (도메인 모델 반환 금지)
- OutPort 반환 타입: **도메인 모델** (Query DTO는 예외)
- DTO 파일 분리: `Command.kt`, `Result.kt`, `Query.kt`, `Spec.kt` — 역할별로 모음
- 모든 DTO는 `data class`

### JPA Entity (`jpa-entity`)
- 위치: `adapter/{도메인}/jpaOut/{entity, repository, service, mapper}/`
- ⚠️ **`@Table(name=..., comment="한글 설명")`, `@Column(comment="한글 설명")` 필수**
- `@org.hibernate.annotations.Comment` 사용 금지 (Hibernate 7 deprecated)
- OutPort 구현체: `@Service` 어노테이션, `adapter/{도메인}/jpaOut/service/*JpaAdapter.kt`

### 매퍼 (`mapper`)
- JPA 매퍼 (`adapter/{도메인}/jpaOut/mapper/`): **확장함수만** (object 싱글톤 금지)
- REST 매퍼 (`adapter/{도메인}/restIn/mapper/`): **MapStruct + 확장함수 공존**
  - `Request → Command`: MapStruct
  - `Result → Response` 단순 1:1: MapStruct
  - `Result → Response` 가공 필요: 확장함수
- Application 매퍼 (`application/{도메인}/mapper/`): Domain → Result 확장함수

### REST Controller (`rest-controller`)
- 위치: `adapter/{도메인}/restIn/controller/`
- **InPort 인터페이스로 의존** (UseCase 구현체 직접 의존 금지)
- Request/Response DTO는 `adapter/{도메인}/restIn/dto/` 위치

### 코딩 컨벤션 (`code-convention`)
- ⚠️ **Boolean에 `is`/`has` 접두사 금지** (`isActive` ❌ → `active` ✅)
- 자기 도메인 식별자는 그대로 `id`, 외부 도메인 참조는 `{도메인}Id` (`memberId`, `socialIdentityId`)
- ktlint: `./gradlew ktlintFormat`, pre-commit hook 자동 실행

### 테스트 (`testing`)
- **Kotest BehaviorSpec** 강제 (FunSpec/JUnit 금지)
- Given/When/Then 구조
- `@SpringBootTest` + `@ActiveProfiles("test")` + `override fun extensions() = listOf(SpringExtension)`
- **OutPort만 모킹** (Repository/Adapter 직접 모킹 금지)
- **MockK 사용** (Mockito 금지)
- 테스트 데이터: **Instancio**
- HTTP 파일: `src/test/resources/http/{도메인}/{파일명}.http`, `@baseUrl`/`@token` 공통 변수

### Gradle (`setup-gradle`)
- 모든 라이브러리 버전: `gradle/libs.versions.toml`에서 중앙 관리
- `build-logic/build.gradle.kts` Versions 객체와 `libs.versions.toml` 동기화 필수
- 사용자 결정: 의존성은 루트 `build.gradle.kts`에 직접 추가 (모듈 분리 시 컨벤션 플러그인으로 이동)
- JPA 사용 모듈: `plugins { id("app"); id("jpa") }`
